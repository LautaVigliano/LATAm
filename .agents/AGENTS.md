# Reglas de Proyecto: LATAm

## Protección de la Rama Principal (main)
1. **Doble Verificación en GitHub:** La rama está protegida con requerimiento de 2 aprobaciones en GitHub.
2. **Flujo de Trabajo:** Todo desarrollo se hace en ramas secundarias (ej. `testing`).
3. **Recordatorio al Usuario:** Al crear un Pull Request hacia `main`, el agente no debe bloquearse innecesariamente; simplemente debe **recordarle al usuario** que vaya a GitHub para aprobar o rechazar la solicitud.
